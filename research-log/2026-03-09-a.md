# Torus CA — Research Log

## System description

A 2D cellular automaton on a toroidal grid. Each node carries an RGB color
treated as a three-component state vector. Directed edges between neighbors
encode the local relationship type (red, green, blue, or null). At each step,
one node is selected at random (asynchronous update). That node competes with
each of its four von Neumann neighbors on the shared edge:

- **Same dominant channel**: both nodes are reinforced — dominant channel
  incremented by `neighbor_step` (8) on both sides. Edge is marked with their
  shared type.
- **Different dominant channels**: the weaker node (smaller dominance margin)
  is converted toward the stronger. Edge is marked null.

After edge competition, if enough surrounding edges agree on a type
(`reinforce_min` or more out of 4), the node self-reinforces in that channel.
With probability `1/mutation_rate`, the node is reset to a uniformly random
color. Dynamics play out on a discrete toroidal grid of size W×H.

**Control parameters**

| Parameter | Role |
|---|---|
| `mutation_rate` (N) | 1-in-N mutation per node selection; drives disorder |
| `reinforce_min` (2–4) | neighbors required to agree for self-reinforcement; sets cooperation threshold |

**Order parameter**: boundary density — fraction of edges whose endpoint nodes
have different dominant channels, scaled to [0, 1].

## What we built

### C simulation (`sim.c`, `sim.h`, `stats.c`, `stats.h`, `main.c`)

Refactored from a monolithic proof-of-concept into a modular library:

- `sim.h` — types (Node, Edge, Torus, Rules), inline color helpers, xorshift64
  RNG with splitmix64 seed mixing
- `sim.c` — `torus_init`, `torus_free`, `compete_edge`, `reinforce_center`,
  `process_node`
- `stats.h/c` — `compute_stats`: boundary density via edge scan, domain count
  and size distribution via BFS flood fill, per-type fractions, log2-binned
  size histogram
- `main.c` — CLI with `--headless`, `--stats-interval`, `--mutation-rate`,
  `--frames`; live progress to `/dev/tty` (visible even when stderr redirected);
  raw RGB24 video to stdout for piping to ffplay/ffmpeg

Compiled with `-O3 -march=native`.

### Python analysis layer

- `analysis/analyse.py` — single-run analysis: equilibrium detection via
  rolling mean, 6-panel plot of all statistics over time
- `analysis/sweep.py` — parameter sweep: parallel subprocess runner
  (ProcessPoolExecutor), CSV aggregation over seeds, 2-panel phase diagram
  plot (bd mean ± seed spread; within-run variance), auto-saves to
  `results/sweep_{W}x{H}_f{frames}_s{seeds}_{timestamp}.{csv,png}`

## What we found

### Sweep 1 — broad scan (mr=50–2000, 128×128, 20k frames, 5 seeds)

Established that boundary density decreases monotonically with mutation rate.
System is clearly ordered at low mutation rates (bd ≈ 0.025 at mr=2000) and
disordered at high mutation rates. Transition region somewhere below mr=150.

Results: `results/sweep_128x128_f20000_s5_20260309_170758.{csv,png}`

### Sweep 2 — transition region (mr=5–150, 128×128, 30k frames, 5 seeds)

Narrowed the transition. bd rises sharply below mr≈20. Variance peaks visible
but noisy. Suggested zooming into mr=4–15.

Results: `results/sweep_128x128_f30000_s5_20260309_185807.{csv,png}`

### Sweep 3 (contaminated) — fine scan (mr=4–15)

Aborted. Two RNG bugs were discovered and fixed mid-run, invalidating results
for power-of-2 mutation rates. Killed before completion.

### RNG bugs discovered and fixed

**Bug 1 — seed collision.** `rng_seed(0)` and `rng_seed(1)` both produced
`g_rng = 1`, giving identical simulation sequences. Fixed by running the seed
through splitmix64 mixing before storing:

```c
seed += 0x9e3779b97f4a7c15ULL;
seed = (seed ^ (seed >> 30)) * 0xbf58476d1ce4e5b9ULL;
seed = (seed ^ (seed >> 27)) * 0x94d049bb133111ebULL;
g_rng = seed ^ (seed >> 31);
if (!g_rng) g_rng = 1;
```

**Bug 2 — low-bit bias.** `rng_next() % n` samples only the low bits of the
xorshift64 output. For LFSR-type generators, low bits have weaker statistical
properties; for power-of-2 values of n (n=8 for mutation_rate=8, the grid
index uses n=16384) the modulo reduces to a simple bit mask. This inflated
boundary density at mr=8 specifically (bd≈0.24 vs expected ≈0.10), producing
a spurious discontinuity that looked like a first-order transition.

Fixed by using the top 32 bits:

```c
static inline size_t rng_range(size_t n) {
    return (size_t)((rng_next() >> 32) % (uint64_t)n);
}
```

### Sweep 4 — clean fine scan (mr=4–15, 128×128, 50k frames, 8 seeds)

**The definitive result at this stage.**

Results: `results/sweep_128x128_f50000_s8_20260310_001608.{csv,png}`

| mr | prob (1/mr) | bd rm=2 | bd rm=3 | bd rm=4 |
|---|---|---|---|---|
| 4  | 0.250 | 0.363 | 0.374 | 0.388 |
| 5  | 0.200 | 0.197 | 0.206 | 0.229 |
| 6  | 0.167 | 0.143 | 0.147 | 0.157 |
| 7  | 0.143 | 0.117 | 0.119 | 0.125 |
| 8  | 0.125 | 0.098 | 0.100 | 0.107 |
| 9  | 0.111 | 0.086 | 0.089 | 0.093 |
| 11 | 0.091 | 0.076 | 0.076 | 0.080 |
| 13 | 0.077 | 0.064 | 0.065 | 0.068 |
| 15 | 0.067 | 0.058 | 0.060 | 0.063 |

**Key findings:**

1. **The transition is continuous.** Boundary density rises smoothly and
   monotonically with mutation probability. The apparent first-order signal
   from sweep 3 was entirely an artifact of the low-bit RNG bias at mr=8.
   This is consistent with the voter model universality class, which predicts
   a continuous transition.

2. **The critical region is near mutation probability 0.15–0.25 (mr=4–7).**
   The steepest slope in bd vs prob falls in this range. Variance peaks
   (from the phase diagram plot) are located here.

3. **reinforce_min=4 shows the sharpest variance peak.** Counterintuitively,
   stricter reinforcement produces a more pronounced critical signal rather
   than smoothing it. This may reflect that strict reinforcement creates
   sharper domain boundaries, amplifying fluctuations at the transition.

4. **reinforce_min has little effect in the ordered phase.** All three curves
   converge at low mutation rates (large mr). The system finds large stable
   domains regardless of reinforcement strictness. reinforce_min matters most
   in the disordered or critical regime.

5. **Large seed-to-seed spread at low mutation rates.** bd_mean_std is large
   at mr=11–15 relative to the mean, suggesting either long equilibration
   times (some seeds still coarsening at 50k frames) or genuine metastability.

## What we are about to do

### Finite-size scaling analysis

The continuous transition opens up a standard finite-size scaling (FSS)
analysis. Near a continuous phase transition, the variance (susceptibility)
peak scales with system linear dimension L as:

```
V_max(L)  ~  L^(γ/ν)
mr_c(L)   ~  mr_c(∞) + A · L^(-1/ν)
```

where γ and ν are critical exponents. By running the same sweep at multiple
system sizes and tracking how the peak height and peak position shift, we can
extract the exponent ratio γ/ν and the correlation length exponent ν.

**Plan:**

1. **64×64** — fast anchor point (minutes per sweep); establishes that the
   peak shifts and broadens as expected
2. **128×128** — done (sweep 4 above)
3. **256×256** — primary new measurement; 4× more nodes, peak should sharpen
   significantly; target ≈15k frames (same total node-updates as 128×128/50k)

Sweep parameters for 256×256:
- Mutation rates: mr=3,4,5,6,7,8 (brackets the critical region)
- reinforce_min: 2, 3, 4
- Seeds: 12
- Workers: 4

After the three sizes are complete, fit V_max(L) vs L on a log-log plot. The
slope is γ/ν. If the system falls near the 2D Ising universality class,
expect γ/ν ≈ 1.75. If it falls near directed percolation, expect different
values. The exponents will identify or rule out candidate universality classes.

**The continuous transition is more useful than a first-order one would have
been for this analysis.** Continuous transitions have universal scaling
behavior with measurable exponents. First-order transitions are
comparatively featureless and do not admit this kind of classification.
